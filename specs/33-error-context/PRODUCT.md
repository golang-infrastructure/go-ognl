# Issue #33: Redacted error context

## Summary

Resolution failures report the object type and numeric selector location that actually failed while keeping selectors, keys, and object values confidential. The same bounded context applies across `Get`, `GetE`, and their `Result` methods without changing the public API.

## Behavior

1. **B01 — Actual failure type.** A resolution failure identifies the dynamic type of the object on which the failing lookup, index, or expansion was attempted, not the root input type or a previously resolved type. A nil current object with no dynamic type uses the exact encoded type token `"<nil>"`; a typed nil reports its encoded dynamic type, not `"<nil>"` or the root type.
2. **B02 — Stable location fields.** A contextual error has one prefix in the exact field order `type=<encoded-type>,offset=<byte-offset>,op=<operation-index>,total_len=<selector-byte-length>: <sentinel-message>`, where `<encoded-type>` follows B01 and B05. `offset`, `op`, and `total_len` are unsigned decimal, zero-based where applicable, and are measured against the original selector supplied to that public call.
3. **B03 — Operation and offset accounting.** Each key lookup, numeric index lookup, and unescaped `#` is one operation; separators and escape bytes do not add operations. An operation's `offset` is always its first source byte in the original selector. If the operation itself begins with an escape, that first source byte is the backslash; a later escape within the same operation does not change the offset. Counting is by selector bytes, not Unicode code points.
4. **B04 — Confidential inputs.** Error strings and `Diagnosis()` entries never contain the selector, a decoded or encoded key, or any traversed object value. This applies to successful prefixes, failing segments, branch values, secrets, and values implementing `fmt.Stringer` or `error`.
5. **B05 — Safe bounded type token.** Every non-nil type field is an ASCII-safe quoted token: quotes, backslashes, control bytes, and non-ASCII bytes are escaped. Together with the exact nil token from B01, every complete encoded type value is at most 256 bytes and may be shortened further only to satisfy B06; truncation preserves valid escaping and includes the literal marker `<truncated>` within the applicable limit.
6. **B06 — Bounded internal errors.** For every error whose unwrap chain terminates at a package sentinel, the complete `Error()` string is at most 352 bytes, remains valid UTF-8, and never ends with a partial escape sequence. When the whole-message cap requires additional shortening, only the encoded type token is shortened; it remains quoted, validly escaped, and marked `<truncated>`, while numeric metadata, the sentinel message, and sentinel identity remain intact.
7. **B07 — Get/GetE format parity.** The fatal error returned by `GetE` and the corresponding entry returned by `Get(...).Diagnosis()` use the same context fields, encoding, redaction, bounds, and sentinel message for the same failure.
8. **B08 — Sentinel identity.** Context wrapping preserves the original package sentinel in the unwrap chain, so `errors.Is` succeeds exactly as it did before contextual formatting was added.
9. **B09 — Result method parity.** `Result.Get` and `Result.GetE` follow B01–B08 using the selector supplied to that method call as the original selector; prior calls do not leak selector text or location state into the new call.
10. **B10 — Deployed branch anchoring.** A failure below an expanded/deployed value retains its original selector byte offset and operation index through branch recursion and receives exactly one context prefix; parent collection does not reset, add, or wrap that context again.
11. **B11 — Partial deployed failure.** When at least one deployed branch resolves, successful values remain available and each failed branch contributes one B01–B10-compliant diagnosis; `GetE` does not turn those branch-local failures into a fatal call error.
12. **B12 — Complete deployed failure.** When every deployed branch fails, the result is ineffective and exposes no partial value. All branch diagnoses remain B01–B10-compliant and appear in input order; the fatal context returned by `GetE` is the context of the first failed branch in that same input order.
13. **B13 — Hostile selector safety.** Valid escaped segments, Unicode, control bytes, selectors up to 1 MiB, and ordinary terminal missing-resolution failures obey B01–B12 without panic or secret disclosure.
14. **B14 — API compatibility.** Issue #33 adds no exported type, function, method, option, context parameter, or configuration surface and does not change existing public signatures.
